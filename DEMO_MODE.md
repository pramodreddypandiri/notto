# Demo Mode - Testing Without Backend

## What is Demo Mode?

Demo Mode allows you to test the full UI/UX of the app without needing to configure Supabase, Claude API, or any backend services. It uses hardcoded sample data to simulate the app's behavior.

## Current Status

âœ… **Demo Mode is ENABLED** by default in both screens.

## How It Works

### Home Screen (Voice Notes)
- **Sample notes** are displayed automatically (bowling, Mexican food, email Jack)
- **Voice recording** works - you can record audio
- **New notes** are added to the list when you stop recording (demo data, not saved)
- No backend calls are made

### Plans Screen (Weekend Plans)
- Click **"Generate Plans"** to see sample weekend plans
- Two demo plans are shown:
  - Saturday Evening Adventure (bowling + Mexican food + dessert)
  - Sunday Afternoon Chill (bowling + tacos)
- **Feedback buttons** work and show confirmation (not saved)
- 1.5 second simulated loading delay for realism

### Settings Screen
- All UI elements are functional
- Location can be typed in (not saved in demo mode)
- No authentication required

## Switching to Real Backend

When you're ready to connect to the real backend:

### Step 1: Configure Backend Services
1. Set up Supabase credentials in `config/supabase.ts`
2. Add Claude API key in `services/claudeService.ts`
3. Set up authentication (create auth screens)

### Step 2: Disable Demo Mode
Change both files:
```typescript
// In app/(tabs)/index.tsx
const DEMO_MODE = false; // Change to false

// In app/(tabs)/plans.tsx
const DEMO_MODE = false; // Change to false
```

### Step 3: Test Real Integration
- Users will need to sign up/login
- Notes will be saved to Supabase
- Plans will be generated by Claude AI
- All data will persist

## What Still Shows Errors?

When demo mode is ON, you'll still see these console warnings/errors (they're safe to ignore):

- `Failed to get notes` - Normal, we're using demo data instead
- `Failed to parse note` - Normal, no Claude API calls in demo mode
- `Not authenticated` - Normal, no auth required in demo mode
- `SafeAreaView deprecated` - Expo SDK update needed (non-critical)
- `expo-av deprecated` - Should migrate to expo-audio (future task)

These errors don't affect functionality in demo mode!
